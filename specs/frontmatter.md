# Frontmatter Specification
<!-- spec: frontmatter -->

Every wiki page MUST have YAML frontmatter. This is how the system tracks, indexes, and manages pages.

---

## Universal Fields (All Page Types)

```yaml
---
type: entity | concept | source | comparison | episode | decision | tradeoff | migration | meta
confidence: VERIFIED | INFERRED | ASSUMED | CONTRADICTED
created: 2026-06-14
updated: 2026-06-14
sources:
  - src/middleware/auth.ts
  - docs/api.md
tags:
  - authentication
  - security
---
```

| Field | Required | Description |
|-------|----------|-------------|
| `type` | ✅ | Page type — determines which memory system it belongs to |
| `confidence` | ✅ | How reliable this knowledge is |
| `created` | ✅ | When the page was first created |
| `updated` | ✅ | When the page was last modified |
| `sources` | ✅ | Raw source files this page was derived from (can be empty) |
| `tags` | ❌ | Freeform tags for categorization |

## Freshness Tracking (Optional)

Any page can include freshness metadata:

```yaml
freshness:
  last_checked: 2026-06-14
  staleness: LOW | MEDIUM | HIGH | STALE
  auto_recheck: true
  check_again: 2026-06-21
```

| staleness | Meaning |
|-----------|---------|
| `LOW` | Checked within freshness_window |
| `MEDIUM` | Checked within 2x freshness_window |
| `HIGH` | Checked within 4x freshness_window |
| `STALE` | Not checked for longer than 4x freshness_window |

## Confidence Level Definitions

| Level | Definition | When to Use | In Answers |
|-------|-----------|-------------|------------|
| `VERIFIED` | Confirmed by code, tests, or user | Ran the code, read the test, user confirmed | State as fact |
| `INFERRED` | Deduced from patterns | Read code but didn't run it, pattern matching | "Based on the code, it appears..." |
| `ASSUMED` | Educated guess | Limited evidence, extrapolation | "I believe... (not confirmed)" |
| `CONTRADICTED` | Conflicts with other evidence | Multiple sources disagree | "There's conflicting information..." |

**Promotion rules:**
- ASSUMED → INFERRED: When code evidence is found
- INFERRED → VERIFIED: When tests pass or user confirms
- Any → CONTRADICTED: When conflicting evidence found
- CONTRADICTED → VERIFIED: When conflict is resolved by user
